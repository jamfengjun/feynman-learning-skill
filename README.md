# Feynman Learning Skill —— 费曼学习法智能助手

> 如果你不能用简单的话讲清楚，说明你还没有真正理解；如果你不能讲给具体对象听，说明它还不能转化为行动。

## 这是什么？

`feynman-learning` 是一个 AI Agent Skill，专为学习、研究、演讲表达和商务谈判设计。它基于费曼学习法，把复杂主题拆成「明确目标 -> 白话解释 -> 暴露缺口 -> 重构输出」四步，帮助你把模糊的信息变成可理解、可验证、可表达、可使用的成果。

它不是单纯的学习笔记模板，而是一个清晰度引擎：只要你需要把一个复杂问题讲明白、研究透、说服别人或准备谈判，都可以调用它。

## 核心特性

- 费曼解释引擎：把复杂概念讲给聪明但不熟悉该领域的人听。
- 知识缺口扫描：识别讲不清、证据不足、定义不稳和假设过强的部分。
- 学习理解模板：输出一句话本质、通俗解释、关键概念、最小例子和自测问题。
- 研究拆解框架：形成研究问题、证据地图、竞争观点、待验证事项和初步结论。
- 演讲表达结构：生成开场钩子、解释阶梯、故事类比、章节大纲和可能问答。
- 商务谈判准备：梳理我方目标、对方关切、价值主张、反对意见、让步顺序和下一步话术。
- 事实边界控制：不虚构客户案例、收入、合作、证书或承诺，未证实内容会标记为假设或待验证。
- 防幻觉协议：区分事实、解释、假设和待验证问题；遇到医学、法律、金融、政策、公司信息、最新技术时优先查权威来源。
- 专业领域适配：面对计算机科学、医学、医疗 AI 等交叉问题，会拆成机制、证据、适用边界、风险和验证路径。

## 快速开始

### 安装

将 `feynman-learning` 文件夹放入你的 Codex Skills 目录即可，例如：

```bash
~/.codex/skills/feynman-learning
```

当前文件结构：

```text
feynman-learning/
├── SKILL.md             # Skill 核心指令
├── README.md            # 项目说明
└── agents/
    └── openai.yaml      # Codex UI 元数据
```

### 使用

对 Agent 说：

```text
用 feynman-learning 帮我理解「大模型微调」。
我有一点技术背景，但不是算法工程师，希望能讲给业务团队听。
```

Agent 会按费曼学习法输出通俗解释、关键概念、最小例子、容易误解的地方和后续需要补齐的问题。

## 更多用法

### 学习理解

```text
用费曼学习法帮我理解「反向传播算法」，要求能讲给非技术合伙人听。
```

### 研究主题

```text
用 feynman-learning 帮我研究「AI 舌诊在慢病管理中的应用价值」，请列出已知事实、待验证问题和初步结论。
```

### 演讲表达

```text
帮我把「糖尿病闭环管理」整理成一个 10 分钟演讲，用费曼法保证外行也能听懂。
```

### 商务谈判

```text
用 feynman-learning 帮我准备一次合作谈判。对方是医院信息科，我方希望推进 AI 健康管理试点。
```

## 工作方式

`feynman-learning` 每次会优先完成四个动作：

1. 明确主题、对象、受众和预期输出。
2. 用白话解释核心概念，减少空泛术语。
3. 暴露知识缺口、证据缺口、逻辑跳跃和潜在反例。
4. 根据场景重构成学习笔记、研究地图、演讲大纲、谈判简报或可直接使用的话术。
5. 对高风险或会变化的信息进行验证，输出置信度和待补证据。

## 防幻觉机制

为了避免把不确定内容包装成确定结论，skill 会默认执行以下规则：

- 把关键信息分成「事实、解释、假设、待验证」四类。
- 优先使用用户提供的原始材料；需要外部事实时，优先使用官方文档、指南、论文、标准、监管文件或公司一手资料。
- 不编造客户案例、营收、合作、证书、临床结果、政策条款、论文结论或技术细节。
- 对研究和决策类输出标注置信度，并说明证据强弱。
- 对「显著提升」「行业领先」「临床验证充分」「安全可靠」这类模糊表述追问指标、对照组、适用人群、标准和来源。

## 案例模式

### 计算机科学

如果解释「梯度下降」，skill 不会只说“优化算法”，而会先用“在雾里下山”的类比解释，再补充输入输出、学习率、局部最优、失败场景和最小例子。

### 医疗 AI

如果研究「AI 舌诊能否用于糖尿病管理」，skill 会区分：

- 可以讲清楚的：辅助观察体质和风险信号。
- 不能夸大的：不能替代血糖、HbA1c、医生诊断。
- 必须验证的：目标人群、样本量、临床终点、对照方法。
- 对外口径：辅助筛查、趋势观察、医生复核。

### 商务谈判

如果准备医院 AI 试点谈判，skill 会把“AI 很先进”重构为“能否降低随访负担、提前发现风险、符合数据安全和临床边界”，再形成试点指标和下一步话术。

## 适用场景

- 学新概念、新行业、新技术、新理论。
- 阅读论文、书籍、课程材料后做消化。
- 准备公开演讲、课程分享、投资人路演或访谈回答。
- 准备销售、合作、采购、融资、内部汇报等商务沟通。
- 把专业内容讲给非专业听众。
- 检查一段论证是否真的讲得清楚。

## 不适用场景

- 需要凭空生成事实、案例、合同、证书、营收或合作关系。
- 需要替代专业法律、医疗、财务结论。
- 只需要简单翻译、格式整理或机械摘要的任务。
